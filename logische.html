Logische laag

Operating System

De processor, ofwel CPU (Central processing unit) is wellicht wel het belangrijkste onderdeel van iedere computer. Echter kan de CPU niet werken zonder andere componenten. Zie de CPU als het brein van de computer; Net als ons brein stuurt het onze overige componenten aan. De CPU ontvang vanuit het werkgeheugen (RAM) instructies en ontvangt deze in de control unit. Deze zoekt de opslagplaats van het benodigde stukje data uit het geheugen van de computer. Deze data wordt doorgestuurd naar de ALU (arithmetic/logic unit)van de processor. Hier word de opdracht verwerkt in binaire codes, door middel van kleine schakelingen die vliegensvlug aan en uit gaan om een 0 of 1 te produceren. Deze binaire instructies sturen de benodigde componenten aan om de taak uit te voeren.
Dit alles gebeurt vliegensvlug. Hoe snel dit is kun je aflezen uit de kloksnelheid van de processor. Deze wordt vaak gegeven in Hertz. Één Hz komt overeen met één bewerking per seconde. Dus de kloksnelheid is hoe snel de CPU een binaire waarde van 0 of 1 kan aangeven, door middel van een kleine schakeling. De meeste moderne CPU’s werken met 3 tot 5 GHz. Dit zijn dan wel 3 tot 5 miljard(!) bewerkingen per seconde. Dit resulteert erin dat computers hele complexe programma’s kunnen runnen op hoge snelheden.
Een andere manier om om CPU snelheid te testen is het gebruiken van MIPS. Officieel staat dit voor ‘millions of instructions per second’. Echter is de MIPS een ruwe weergave van de snelheid van de CPU, omdat deze sterk afhangt van welke soort instructies de CPU moet verwerken. Daarom wordt MIPS ook wel eens vertaald naar ‘Meaningless indication of processor speed’.

DBMS

RAM is een acroniem voor ‘Random Access Memory’. In de RAM wordt de data opgeslagen die de CPU op dat moment nodig heeft. Data in de RAM is broodnodig, omdat deze veel sneller op te halen valt dan de data uit de harde schrijf.
Bij het opstarten van je computer is de dataopslag van de RAM initieel leeg. De RAM vult telkens zichzelf met de benodigde data vanuit andere opslag bronnen waarmee de CPU de benodigde taak kan uitvoeren. Dus als jij je browser opent, verzamelt de RAM alle data waarmee de CPU alle taken kan voltooien. Dit systeem is bedacht door John von Neumann.
Zie de RAM dus als je het tafelblad van je bureau, waar je alle dingen neerlegt die je op dat moment nodig hebt; Denk aan een leeslamp, je favoriete boek en een pennenbakje. Naast je bureau staat een ladekast met alle papieren die je op korte termijn niet verwacht te gebruiken. Dit is dan jouw harde schrijf.
De grootte van de RAM opslag heeft invloed op de snelheid. Hoe meer GB werkgeheugen je in je computer hebt, des te sneller je programma's kunnen laden. Met meer werkgeheugen is het ook beter mogelijk om meerdere programma's tegelijk open te hebben.
ROM staat voor “Read-Only-Memory” en is in tegenstelling tot de RAM een opslagplaats die permanente data vasthoud; Namelijk de data die je computer nodig heeft om op te starten en te functioneren. Denk hierbij aan de data van de OS. Deze files blijven gedurende de levensloop van de computer onveranderd en kunnen door de gebruiker niet worden aangepast (vandaar ‘Read-Only’). .
